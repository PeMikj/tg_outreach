# Governance & Risk Management

## 1. Реестр рисков

| Риск | Вероятность | Влияние | Детектирование | Митигирование | Остаточный риск |
|------|-------------|----------|----------------|----------------|-----------------|
| Блокировка аккаунта | Средняя | Высокое | Ошибка API | Лимиты + backoff | Низкий |
| Спам-поведение | Средняя | Высокое | Анализ response rate | Жёсткая policy | Низкий |
| Дубликат сообщений | Средняя | Среднее | Memory check | Уникальность рекрутера | Очень низкий |
| Утечка данных | Низкая | Критическое | Аудит | Локальное шифрование | Низкий |
| Галлюцинация LLM | Средняя | Среднее | Валидация длины | Шаблоны | Низкий |
| Сообщение после отказа | Низкая | Высокое | Статус диалога | Policy stop rule | Очень низкий |
| Prompt injection | Средняя | Среднее | Контент-фильтр | Структурированный parsing | Низкий |

---

## 2. Политика работы с персональными данными

- Данные хранятся локально
- Шифрование at-rest
- Логи не содержат персональных идентификаторов
- Нет передачи данных третьим сторонам

---

## 3. Логирование

- Структурированные логи
- Маскирование username
- Отдельный audit log
- Нет хранения полного CV в логах

---

## 4. Защиты

- Подтверждение перед первым outreach
- Ограничение количества сообщений
- Не более 1 follow-up
- Остановка при отказе
- Emergency stop
- Cooldown при предупреждении Telegram

---

## 5. Защита от injection

- LLM изолирован от system prompt
- Нет выполнения текста как команды
- Ответы рекрутера не интерпретируются как инструкции системе
